## Phase 3: 会话管理 + 追问

### 任务 3.1 — 新增 `backend/session_service.py`

- [ ] 创建文件，实现 `class SessionManager`:
  - `get_or_create_session(session_id: str) -> dict`
    - 查 `chat_sessions` 表，不存在则 INSERT，返回 row dict
    - 新建时 `title='新对话'`, `status='active'`, `context_json='{}'`
  - `add_message(session_id, role, content, message_type='text', metadata=None) -> None`
    - INSERT 到 `chat_messages`
    - UPDATE `chat_sessions` 的 `message_count += 1`
    - 如果是首条用户消息，用消息前 20 字更新 `title`
  - `get_history(session_id, limit=20) -> list[dict]`
    - 按 `created_at ASC` 返回最近 limit 条消息
  - `update_context(session_id, context: dict) -> None`
    - UPDATE `chat_sessions.context_json`
  - `list_sessions(session_ids: list[str] = None) -> list[dict]`
    - 按 `updated_at DESC` 返回会话列表
    - 如果传了 session_ids，只返回这些 ID 的会话
  - `archive_session(session_id) -> None`

- [ ] 实现 `class FollowUpManager`:
  - `should_follow_up(intent_result: IntentResult, context: dict) -> bool`
    - `slots_missing` 非空且 `follow_up_count < 3` → True
  - `generate_follow_up(intent_result, context, user_intent) -> dict`
    - 根据 `slots_missing` 和 `user_intent` 生成追问
    - 返回 `{"question": str, "options": list[str], "missing_slots": list[str]}`
  - `merge_slot_reply(context: dict, user_reply: str, intent_result: IntentResult) -> dict`
    - 将本轮槽位合并到 `context.slots`
    - 重置 `slots_missing`/`slots_ambiguous`
    - 递增 `follow_up_count`

## Phase 4: 动态规划 + 编排

### 任务 4.1 — 新增 `backend/plan_engine.py`

- [ ] 创建文件，定义 `PlanStep` 和 `Plan` 数据类

- [ ] 实现 `class PlanEngine`:
  - `build(intent, slots, context, user_intent) -> Plan`
    - 查决策表（见下方），组装 Plan
  - **决策表实现**（硬编码，不用 LLM）:
    ```
    product_selection + 槽位齐全     → [fetch_candidates, call_all_agents, score, report]
    product_selection + 槽位不全首轮 → [follow_up]
    product_selection + 槽位不全≥2轮 → [quick_search, format_top3, follow_up_confirm]
    product_selection + 候选<3       → [expand_criteria, re_search, follow_up_confirm]
    report_lookup                    → [query_reports, format_report_card]
    product_compare                  → [lookup_products, call_market+review, comparison_report]
    supply_inquiry                   → [call_supply_agent, supply_report]
    general_chat                     → [llm_chat_reply]
    ```
  - **fallback**: 任意异常 → 友好提示 + 建议重新描述需求

- [ ] 实现 `class PlanExecutor`:
  - `async execute(plan: Plan) -> PlanResult`
    - 按 `depends_on` 的顺序执行 steps
    - 每个 step 根据 `step_type` 走不同逻辑：
      - `"query"` → 调用 `query_candidate_products()` 或查 DB
      - `"agent_call"` → 调 `call_a2a_agent(agent_name, ...)`
      - `"score"` → 调 `calculate_final_score()`
      - `"report"` → 调 `generate_report()`
      - `"follow_up"` → 返回追问（由上层处理）
    - step 失败且 `retry_on_fail=True` → 重试 1 次
    - 返回 `PlanResult`（包含 products, report, error 等）

### 任务 4.2 — 新增 `backend/session_orchestrator.py`

- [ ] 创建文件，实现 `async def handle_message(request: ChatRequest) -> ChatResponse`:
  1. `session = SessionManager().get_or_create_session(request.session_id)`
  2. `history = SessionManager().get_history(request.session_id, limit=10)`
  3. `user_intent = UserIntentRecognizer().recognize(request.message, history)`
  4. `intent_result = IntentClassifier().classify(request.message)`
  5. 如果 `FollowUpManager().should_follow_up(intent_result, session.context)`:
     - 生成追问 → 返回 `ChatResponse(follow_up=...)`
  6. 否则 `plan = PlanEngine().build(...)` → `result = await PlanExecutor().execute(plan)`
  7. 更新会话（add_message × 2, update_context）
  8. 返回 `ChatResponse(message=..., result=...)`
